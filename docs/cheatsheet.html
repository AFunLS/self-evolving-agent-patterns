Production AI Agent Patterns

Extracted from JARVIS — 1,000+ cycles, 500+ self-modifications, zero catastrophic failures

PATTERN 1

🎯 Context Engineering

Control LLM behavior by shaping input, not parsing output. The #1 most impactful pattern.

Give structured tools (report_result with enums)
Parse free text with regex
PATTERN 2

🛡️ Immune System

Every failure → documented anti-pattern → injected into context → permanent immunity.

Document failures in system prompt
Hope the agent "learns" implicitly
PATTERN 3

⚖️ Two-Paradigm Discipline

Mechanical decisions → code. Semantic decisions → LLM. Behavioral changes → context.

LLM evaluates quality
if "success" in response
PATTERN 4

📋 Constitution

Hard rules that can never be violated — even by the agent modifying itself.

YAML rules checked before every write
Trust the LLM to self-constrain
PATTERN 5

🔄 Evolution + Verification

Self-modify → mechanical verify (syntax, tests) → adversarial review (separate LLM) → commit or revert.

Independent LLM reviews diffs
Let agent judge its own changes
PATTERN 6

📊 Purpose Profiles

Different tasks need different context. A scheduler sees budget; a coder sees architecture.

Profile-based context assembly
Same giant prompt for everything
PATTERN 7

🏗️ Artifact-or-Nothing

Every cycle must produce a verifiable artifact. No commit = failed cycle. No exceptions.

End cycle with git commit hash
"Assessed situation" as success
PATTERN 8

🔧 Tool Auto-Discovery

Drop a tool file in the directory → it's auto-registered. No imports, no manual wiring.

Convention-based registration
Manual tool registry editing
PATTERN 9

💰 Budget Tracking

Per-cycle cost tracking with auto-throttle. Without it: $180 burned in 3 hours doing nothing.

Track cost per cycle, enforce limits
Discover costs from the API bill
BONUS

🧠 Reward-Driven Evolution

Track success/failure per strategy. Surface winning patterns. Suppress losing ones. The agent gets smarter over time.

Structured reward logging
Untracked trial-and-error

⚡ Quick Reference: Decision Matrix

Decision Type Use Example
File exists? Test pass? Code os.path.exists(), exit_code == 0
Is this code good? LLM Send diff to separate LLM instance
Change agent behavior Context Edit system prompt, not add if-elif
Collect facts Code → LLM Code gathers data, LLM interprets
Parse LLM output Don't. Use tool calls with typed schemas
